注意，懒熊可不是在说大话，本教程绝对不涉及任何代码操作！
纯新手认真阅读后也能包教包会，五分钟就能搭建一个小站

🐾partI 永硕E盘

步骤一 👉点这里免费注册永硕E盘账号👈

🐾partII 博客搭建（Windows环境）

准备工作

  • 👉点击下载Git👈
  • 👉点击下载Gridea👈
  • 👉点击注册GitHub账户

然后安装你下载的Git和Gridea
没有特殊需求的话，一路点next就好

步骤一 创立一个Github仓库

  • 点击new
  • 点击create repository
  • 创立一个你的GitHub账户的token（令牌）

    1.点开右上角头像
    2.点击settings
    3.下拉找到Developer settings
    4.点击Personal access tokens
    5.点击Generate new token

接着会出现一串字符，请及时保存，因为它只出现一次
这就是你申请到的token（令牌）

步骤二 启动一下你安装好的Git

点击一下git-cmd和git-bash就好
过一会等黑框里有字了，就关闭这个窗口

步骤三 设置Gridea

  • 打开安装好的Gridea
  • 找到远程，做以下工作(忽略懒熊手快打错的单词）
  • 做完上述后，再点击同步（不过懒熊强烈建议你在同步前先进行本地预览），你的博客就做好了

这样就结束了，你获得了一个网址为https://username.github.io 的个人博客！
你可以像懒熊一样在这里发布和小站相关的东西
或者拿博客做一些自己喜欢的东西（存放自己的博文、记录生活、整理喜欢的文章等等）

注：有时候你点过同步后，在电脑上点击username.github.io网址会发现内容完全没同步
你可以使用手机/平板或其他设备打开你的博客网址，在那里其实是同步的，自己电脑好像会有延迟

Q4 GitHub pages的优劣

优势：注册只需邮箱，咱们用到的功能全部免费，你还能免费绑定域名且能自动部署
劣势：国内直连访问速度成迷，反正我的设备全部能直连，基友有时候就不行（所以小伙伴要是偶尔打不开博客，可以切换网络或者等会再试试）

Q5 Gridea的优劣

Gridea是一个静态部署balabala的工具
实话实话，我目前觉得它对我这个小白只有优势，没有劣势
它界面易操作、支持markdown、完全免费、拥有丰富且易于使用的主题
对我来说，它几乎完美，我连同步都没出过错

博客最重要的不就是写作吗？简单的操作，然后专注内容就好
再难点儿的懒熊也不会啊！

Q6 可以不用GitHub pages吗

当然可以，你连博客不搭都可以（我认真的）

好吧，如果你不想用GitHub pages服务还想简易搭建这样一个博客
我推荐你使用gitee pages或者Coding Pages进行搭建，这两个都是国内平台，访问速度嗖嗖的

你可以把gitee简单理解为github的国内版
👉戳我阅读coding pages+Gridea教程👈

Q7 关于gitee pages

你可能已经注意到了，gridea的远程里支持coding pages，却不支持gitee pages，那我为什么还要推荐gitee pages呢？
事实上，gitee是github的国内翻版，你只要搭建一个github pages然后百度同步教程，用git将两个同步就好

✈️这样一来，你就拥有了一个网址为https://username.gitee.io的个人博客，根本不用担心网络问题

Q8 那你为什么还要选择GitHub pages

gitee pages有两个致命的缺点

😕首先他的注册需要手机号、开通pages需要严格的实名认证（这个除了麻烦倒也没什么）
😡可是！gitee pages有可能会不给通知的停掉你的pages，即使内容没有违规，所以我们需要备份
😩剩下就是gitee pages绑定域名需要付费、自动部署也需要付费

coding pages也有同样缺陷

Gitee和GitHub给你生成博客地址都是规律的，可是coding pages不是，他会给很杂乱的二级域名，就因为这个就足够我舍弃不用了

Gridea当然也支持评论系统，你只需要按照教程申请账号填写token即可
